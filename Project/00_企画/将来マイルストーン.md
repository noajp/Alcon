# Alcon 将来マイルストーン

SAP・ServiceNow・Salesforce の全機能を `S/O/E + Tag + Ticket + Widget + Worker` で再解釈したモジュールロードマップ。各モジュールは独立リポ・独立デプロイ、共通基盤を共有するSAPモジュール型アーキテクチャ。

## 共通基盤（全モジュール共有）

```
System  → テナント / 組織 / 事業体
Object  → ドメイン固有の中間構造（部門・プロジェクト・製品ライン）
Element → 最小実行単位（仕訳・案件・インシデント・製品・患者）
Tag     → ドメイン固有の属性を後付け（業界語彙はここで吸収）
Ticket  → 戦略・判断・議論（思考層）
Widget  → KPI / Gantt / Map / Chart（表現層）
Worker  → Human + AI Agent（実行層）
```

## 競合との差分

| 観点 | SAP | ServiceNow | Salesforce | **Alcon** |
|---|---|---|---|---|
| 中心概念 | モノ（在庫・資産） | チケット（リクエスト） | ヒト（顧客） | **何でも（Element）** |
| ドメイン | 製造・財務 | IT運用 | CRM | **全ドメイン** |
| 戦略連携 | 弱い | 弱い | 弱い | **Ticket で戦略〜実行を一本化** |
| UI統一 | モジュール間で断絶 | IT色が強い | CRM色が強い | **全モジュール同一UX** |
| AI統合 | 後付け | 後付け | Einstein（部分） | **Worker として基盤に内包** |
